## Property Details

### createChildContentSections {#createchildcontentsections-propertysignature}

Policy for generating child content sections within a section describing an API item that potentially has children (see [TransformApiItemWithChildren](/docs/apis/api-markdown-documenter#transformapiitemwithchildren-typealias)<!-- -->).

#### Signature {#createchildcontentsections-signature}

```typescript
createChildContentSections?: CreateChildContentSections;
```

#### Remarks {#createchildcontentsections-remarks}

This policy is used by the default policies of many of the other transformation policy options. This can be used to adjust the layout of the child sections for API item kinds that have without having to provide new transformation overrides for all of those content types.

### transformApiModel {#transformapimodel-propertysignature}

Policy for transforming a section describing a `Model`<!-- -->.

#### Signature {#transformapimodel-signature}

```typescript
transformApiModel?: TransformApiItemWithoutChildren<ApiModel>;
```

#### Remarks {#transformapimodel-remarks}

Note that this is a [TransformApiItemWithoutChildren](/docs/apis/api-markdown-documenter#transformapiitemwithoutchildren-typealias) only because we handle `Model` and `Package` items specially. We never render `Package` child details directly to the `Modal` document. These are always rendered to seperate documents from each other.
