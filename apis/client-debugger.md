[Packages](/docs/apis/) &gt; [@fluid-tools/client-debugger](/docs/apis/client-debugger)

Contains a simple API for initializing debug sessions for recording and propogating information about a given <i>Fluid Container</i> and its <i>@fluidframework/container-definitions\#Audience</i>.

Debugger instances are created per `Container` instance and are bound to the `globalThis` context to be accessible to related tools.

- See the package README for more details on related tools that are enabled via these debugger instances.

## Remarks {#_fluid-tools_client-debugger-remarks}

The general usage pattern for this library is to first initialize a debugger for a given Fluid Client (<!-- --><i>@fluidframework/container-definitions\#IContainer</i> and <i>@fluidframework/container-definitions\#IAudience</i>) by calling [initializeFluidClientDebugger()](/docs/apis/client-debugger#initializefluidclientdebugger-function) during application setup / any time after your container has been attached.

Then, during application teardown, call [closeFluidClientDebugger()](/docs/apis/client-debugger#closefluidclientdebugger-function) to clean up the debugger and its resources.

## Examples {#_fluid-tools_client-debugger-examples}

### Example 1 {#_fluid-tools_client-debugger-example1}

Initialization

```typescript
initializeFluidClientDebugger({
 containerId,
 container,
 containerData: {
     rootMap: sharedMap
 },
});
```

### Example 2 {#_fluid-tools_client-debugger-example2}

Disposal

```typescript
closeFluidClientDebugger(containerId);
```

### initializeFluidClientDebugger {#initializefluidclientdebugger-function}

Initializes a <i>IFluidClientDebugger</i> from the provided properties, binding it to the global context.

#### Signature {#initializefluidclientdebugger-signature}

```typescript
export declare function initializeFluidClientDebugger(props: FluidClientDebuggerProps): void;
```

#### Remarks {#initializefluidclientdebugger-remarks}

If there is an existing debugger session associated with the provided [FluidClientDebuggerProps.containerId](/docs/apis/client-debugger\fluidclientdebuggerprops-interface#containerid-propertysignature)<!-- -->, the existing debugger session will be closed, and a new one will be generated from the provided props.
