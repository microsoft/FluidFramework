Declares the Fluid objects that will be available in the [Container](/docs/apis/fluid-static/ifluidcontainer-interface).

## Signature {#containerschema-signature}

```typescript
export interface ContainerSchema
```

## Remarks {#containerschema-remarks}

It includes both the instances of objects that are initially available upon `Container` creation, as well as the types of objects that may be dynamically created throughout the lifetime of the `Container`.

## Property Details

### dynamicObjectTypes {#dynamicobjecttypes-propertysignature}

Loadable objects that can be created after the initial [Container](/docs/apis/fluid-static/ifluidcontainer-interface) creation.

#### Signature {#dynamicobjecttypes-signature}

```typescript
dynamicObjectTypes?: LoadableObjectClass<any>[];
```

#### Remarks {#dynamicobjecttypes-remarks}

Types defined in `initialObjects` will always be available and are not required to be provided here.

For best practice it's recommended to define all the dynamic types you create even if they are included via initialObjects.

### initialObjects {#initialobjects-propertysignature}

Defines loadable objects that will be created when the [Container](/docs/apis/fluid-static/ifluidcontainer-interface) is first created.

#### Signature {#initialobjects-signature}

```typescript
initialObjects: LoadableObjectClassRecord;
```

#### Remarks {#initialobjects-remarks}

It uses the key as the id and the value as the loadable object to create.

#### Example {#initialobjects-example}

In the example below two objects will be created when the `Container` is first created. One with id "map1" that will return a `SharedMap` and the other with id "pair1" that will return a `KeyValueDataObject`.

```typescript
{
  map1: SharedMap,
  pair1: KeyValueDataObject,
}
```
