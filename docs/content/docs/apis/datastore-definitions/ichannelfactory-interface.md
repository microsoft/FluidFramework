Definitions of a channel factory.

## Signature {#ichannelfactory-signature}

```typescript
export interface IChannelFactory
```

## Remarks {#ichannelfactory-remarks}

The runtime must be able to produce "channels" of the correct in-memory object type for the collaborative session. Here "channels" are typically distributed data structures (DDSs).

The runtime will consult with a registry of such factories during [Container](https://fluidframework.com/docs/build/containers/) load and when receiving "attach" operations (ops), which indicate a new instance of a channel being introduced to the collaboration session, to produce the appropriate in-memory object.

Factories follow a common model but enable custom behavior.

## Example {#ichannelfactory-example}

If a collaboration includes a [SharedMap](https://fluidframework.com/docs/data-structures/map/), the collaborating clients will need to have access to a factory that can produce the `SharedMap` object.
