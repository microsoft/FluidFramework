## Method Details

### getRevisionView (ALPHA) {#getrevisionview-methodsignature}

Returns the `TreeView` output associated with the largest revision in `editLog` less than (but not equal to) the supplied revision.

For example:

- revision 0 returns the initialRevision.

- revision 1 returns the output of editLog\[0\] (or initialRevision if there is no edit 0).

- revision Number.POSITIVE\_INFINITY returns the newest revision.

{{% callout warning This API is deprecated and will be removed in a future release. %}}
Edit virtualization is no longer supported, use [getRevisionViewInMemory(revision)](/docs/apis/tree/logviewer-interface#getrevisionviewinmemory-methodsignature)

{{% /callout %}}

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.

### getRevisionViewInMemory (ALPHA) {#getrevisionviewinmemory-methodsignature}

Returns the `TreeView` output associated with the largest revision in `editLog` less than (but not equal to) the supplied revision. Can only be used to retrieve revisions added during the current session that have not been evicted from `editLog`.

For example:

- revision 0 returns the oldest edit in the log (which might be initialRevision).

- revision 1 returns the output of editLog\[0\] (or initialRevision if there is no edit 0).

- revision Number.POSITIVE\_INFINITY returns the newest revision.

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.

#### Signature {#getrevisionviewinmemory-signature}

