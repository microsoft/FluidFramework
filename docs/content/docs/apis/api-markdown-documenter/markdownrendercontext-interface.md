## Property Details

### headingLevel {#headinglevel-propertysignature}

Contextual heading level.

#### Signature {#headinglevel-signature}

```typescript
headingLevel: number;
```

#### Remarks {#headinglevel-remarks}

Will automatically increment based on [SectionNode](/docs/apis/api-markdown-documenter/sectionnode-class)s encountered, such that heading levels can be increased automatically based on content hierarchy.

### insideCodeBlock {#insidecodeblock-propertysignature}

Whether or not we are currently rendering inside of a code block.

#### Signature {#insidecodeblock-signature}

```typescript
readonly insideCodeBlock?: boolean;
```

#### Remarks {#insidecodeblock-remarks}

Textual content within code blocks must not be escaped, in order to be Markdown compatible. We use this flag during rendering to determine whether or not we may escape contents.

### insideHtml {#insidehtml-propertysignature}

Whether or not we are currently rendering as a child of some HTML content, within the Markdown document.

#### Signature {#insidehtml-signature}

```typescript
readonly insideHtml?: boolean;
```

#### Remarks {#insidehtml-remarks}

Any content being rendered in an HTML context must also be rendered as HTML. I.e. Markdown content may contain HTML content, but not vice-versa.

### insideTable {#insidetable-propertysignature}

Whether or not we are currently rendering inside of a table context.

#### Signature {#insidetable-signature}

```typescript
readonly insideTable?: boolean;
```

#### Remarks {#insidetable-remarks}

Certain kinds of Markdown content (namely, multi-line contents) cannot be correctly rendered within a Markdown table cell. To work around this, we render some kinds of child content as HTML when inside of a table cell context.

### renderers {#renderers-propertysignature}

Configuration for rendering different kinds of [DocumentationNode](/docs/apis/api-markdown-documenter/documentationnode-interface)s.

#### Signature {#renderers-signature}

```typescript
renderers: MarkdownRenderers;
```

#### Remarks {#renderers-remarks}

Will include default renderers for all [DocumentationNode](/docs/apis/api-markdown-documenter/documentationnode-interface) types enumerated in [DocumentationNodeType](/docs/apis/api-markdown-documenter#documentationnodetype-enum).
