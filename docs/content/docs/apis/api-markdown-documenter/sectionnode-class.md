## Property Details

### Empty {#empty-property}

Empty section singleton.

#### Signature {#empty-signature}

```typescript
static readonly Empty: SectionNode;
```

### heading {#heading-property}

Optional heading to display for the section.

#### Signature {#heading-signature}

```typescript
readonly heading?: HeadingNode;
```

#### Remarks {#heading-remarks}

If not specified, no heading will be displayed in the section contents. Note that this section will still influence heading hierarchy of child contents regardless.

### singleLine {#singleline-property}

Whether or not the content of the node fits on a single line.

#### Signature {#singleline-signature}

```typescript
get singleLine(): false;
```

#### Remarks {#singleline-remarks}

Certain classes of items are required to be single-line only, and will use this flag to violate child contents, etc.

### type {#type-property}

The type of Documentation domain node.

#### Signature {#type-signature}

### combine {#combine-method}

Merges a list of [SectionNode](/docs/apis/api-markdown-documenter/sectionnode-class)s into a single section.

#### Signature {#combine-signature}

```typescript
static combine(...sections: SectionNode[]): SectionNode;
```

#### Remarks {#combine-remarks}

This is an option if you wish to group a series of sections without putting them under some parent section (which would affect the hierarchy).
