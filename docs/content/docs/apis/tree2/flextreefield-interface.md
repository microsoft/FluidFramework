A collaboratively editable collection of nodes within a [FlexTreeEntity](/docs/apis/tree2/flextreeentity-interface).

Fields are inherently part of their parent, and thus cannot be moved. Instead their content can be moved, deleted or created.

Editing operations are only valid on trees with the _TreeStatus\#InDocument_ `TreeStatus`.

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.

## Signature {#flextreefield-signature}

```typescript
export interface FlexTreeField extends FlexTreeEntity<TreeFieldSchema>
```

**Extends:** [FlexTreeEntity](/docs/apis/tree2/flextreeentity-interface)&lt;[TreeFieldSchema](/docs/apis/tree2/treefieldschema-class)&gt;

## Remarks {#flextreefield-remarks}

Fields are used wherever an editable collection of nodes is required. This is required in two places: 1. To hold the children of non-leaf [FlexTreeNode](/docs/apis/tree2/flextreenode-interface)s. 2. As the root of a [FlexTreeEntity](/docs/apis/tree2/flextreeentity-interface).

Down-casting (via [is(schema)](/docs/apis/tree2/flextreefield-interface#is-methodsignature)) is required to access Schema-Aware APIs, including editing. All content in the tree is accessible without down-casting, but if the schema is known, the schema aware API may be more ergonomic.

### isSameAs (ALPHA) {#issameas-methodsignature}

Check if this field is the same as a different field. This is defined to mean that both are in the same editable tree, and are the same field on the same node. This is more than just a reference comparison because unlike EditableTree nodes, fields are not cached on anchors and can be duplicated.

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.
