[FlexTreeField](/docs/apis/tree2/flextreefield-interface) that stores a sequence of children.

Sequence fields can contain an ordered sequence any number of [FlexTreeNode](/docs/apis/tree2/flextreenode-interface)s which must be of the [AllowedTypes](/docs/apis/tree2#allowedtypes-typealias) from the [TreeFieldSchema](/docs/apis/tree2/treefieldschema-class)).

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.

## Remarks {#flextreesequencefield-remarks}

Allows for concurrent editing based on index, adjusting the locations of indexes as needed so they apply to the same logical place in the sequence when rebased and merged.

Edits to sequence fields are anchored relative to their surroundings, so concurrent edits can result in the indexes of nodes and edits getting shifted. To hold onto locations in sequence across an edit, use anchors.
