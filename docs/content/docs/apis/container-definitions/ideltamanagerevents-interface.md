### (event: "op", listener: (message: ISequencedDocumentMessage, processingTime: number) =&gt; void): any {#_call__2-callsignature}

#### Signature {#_call__2-signature}

```typescript
(event: "op", listener: (message: ISequencedDocumentMessage, processingTime: number) => void): any;
```

#### Remarks {#_call__2-remarks}

Note: this event is not intended for general use. Prefer to listen to events on the appropriate ultimate recipients of the ops, rather than listening to the ops directly on the [IDeltaManager](/docs/apis/container-definitions/ideltamanager-interface).

Listener parameters:

- `message`: The op that was processed.

- `processingTime`: The amount of time it took to process the inbound operation (op), expressed in milliseconds.

Emitted when the [IDeltaManager](/docs/apis/container-definitions/ideltamanager-interface) completes connecting to the Fluid service.

#### Signature {#_call__4-signature}

```typescript
(event: "connect", listener: (details: IConnectionDetails, opsBehind?: number) => void): any;
```

#### Remarks {#_call__4-remarks}

This occurs once we've received the connect\_document\_success message from the server, and happens prior to the client's join message (if there is a join message).

Listener parameters:

- `details`: Connection metadata.

- `opsBehind`: An estimate of far behind the client is relative to the service in terms of ops. Will not be specified if an estimate cannot be determined.
