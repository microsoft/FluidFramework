## Property Details

### checkpointSequenceNumber {#checkpointsequencenumber-propertysignature}

Last known sequence number to ordering service at the time of connection.

#### Signature {#checkpointsequencenumber-signature}

```typescript
checkpointSequenceNumber: number | undefined;
```

#### Remarks {#checkpointsequencenumber-remarks}

It may lap actual last sequence number (quite a bit, if container is very active). But it's the best information for client to figure out how far it is behind, at least for "read" connections. "write" connections may use own "join" op to similar information, that is likely to be more up-to-date.
