### cannotCatchUp (ALPHA) {#cannotcatchup-enummember}

This error will be raised when client is too behind with no way to catch up. This condition will happen when user was offline for too long, resulting in old ops / blobs being deleted by storage, and thus removing an ability for client to catch up. This condition will result in any local changes being lost (i.e. only way to save state is by user copying it over manually)

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.

#### Signature {#cannotcatchup-signature}

```typescript
cannotCatchUp = "cannotCatchUp"
```

### fetchTimeout (ALPHA) {#fetchtimeout-enummember}

Maximum time limit to fetch reached. Host application specified limit for fetching of snapshot, when that limit is reached, request fails. Hosting application is expected to have fall-back behavior for such case.
