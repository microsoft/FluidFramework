Contains an API for initializing developer tooling alongside the Fluid Framework.

This tooling is built for use alongside our Chromium browser extension. It listens for incoming messages from the corresponding extension, and posts direct responses as well as automatic updates for Fluid state changes.

Individual [Fluid Containers](/docs/apis/container-definitions/icontainer-interface) can be registered to generate Container-level stats.

Visualization of telemetry logs can be enabled by creating a _DevtoolsLogger_ and providing it to Devtools initialization.

See the package README for more details.

## Type Details

### ContainerKey (ALPHA) {#containerkey-typealias}

A key used to identify and differentiate Containers registered with the _IFluidDevtools_.

WARNING: This API is provided as an alpha preview and may change without notice. Use at your own risk.

#### Signature {#containerkey-signature}

```typescript
export type ContainerKey = string;
```

#### Remarks {#containerkey-remarks}

Each Container registered with the Devtools must be assigned a unique `containerKey`.
