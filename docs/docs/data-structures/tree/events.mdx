## Whole-Tree Events

`TreeView` exposes two events for tree-wide changes:

- `rootChanged` fires when the root field changes — i.e., a new root node is assigned or the schema changes.
It does not fire when a property of the root node changes.
- `commitApplied` fires when a change is applied outside a transaction, or when a transaction commits.
Use this to obtain `Revertible` objects for undo/redo stacks.
See [undo/redo support](./undo-redo.mdx) and [Transactions](./transactions.mdx).

To subscribe to an event:

```typescript
const unsubscribe = treeView.events.on("rootChanged", () => {...});

// Later at some point when the event subscription is not needed anymore
unsubscribe();
```

## Node-Level Events

`SharedTree` supports two node-level events, subscribed to via `Tree.on`:

```typescript
on<K extends keyof TreeChangeEvents>(
		node: TreeNode,
		eventName: K,
		listener: TreeChangeEvents[K],
	): () => void;
```

`eventName` is either `"nodeChanged"` or `"treeChanged"`:

-   `nodeChanged` fires when one or more properties of `node` change.
-   `treeChanged` fires when one or more properties of `node` or any node in its subtree changes.

See the API docs for each event for further details.

`Tree.on()` returns an unsubscribe function, typically called during cleanup:

```typescript
const unsubscribe = Tree.on(myTreeNode, "nodeChanged", () => {...});

// Later at some point when the event subscription is not needed anymore
unsubscribe();
```
